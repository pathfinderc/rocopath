# 洛寻（rocopath）

## 项目介绍

洛寻是一个**地图路线规划与标注工具**，仅供学习交流使用。

本项目本身是一个功能框架，提供了地图背景叠加、点位标记、路线编辑等基础功能。用户可依据本项目定义的数据格式规范，自行准备数据文件，以实现特定场景下的地图标注辅助。

> **核心设计原则**
>
> 本项目遵守 **代码与数据严格分离** 的原则。代码仓库**不包含**、**不分发**、**不提供**任何游戏厂商的原始数据或二次加工后的数据文件。用户因自行获取、使用相关数据文件所产生的任何问题，与本项目及项目维护者无关。

## 重要声明

1. **非商业用途**：本项目完全免费开源，**不盈利**，**不接受捐赠**，仅用于技术学习与交流。
2. **非外挂声明**：本项目**不是**外挂或脚本，**不会**以任何形式读取、修改游戏客户端内存或与游戏进程交互。项目仅提供静态的地图标注与路线规划功能。
3. **通用工具属性**：本项目不对任何特定游戏的数据来源负责。程序本身是一个地图标注框架，未内置指向任何特定游戏的可运行数据。若有示例配置，仅为说明数据格式之用。
4. **数据可靠性**：目前游戏中 NPC 和采集物的刷新位置可能已改为服务器下发，本项目使用的数据仅为格式示例，可能与实际游戏不匹配，请自行核实。
5. **权利响应**：如权利人认为本项目侵犯其合法权益，请通过 GitHub Issues 联系，我们将在收到通知后及时处理。

## 数据准备

本项目**不提供任何游戏原始数据**，你需要**自行网络搜索**符合本项目格式规范的数据文件，并将其放置在 `assets/` 目录下。

期望的目录结构如下，期望的图片尺寸为8192x8192像素：

```
assets
├── data
│   ├── AREA_CONF.json
│   ├── NPC_CONF.json
│   ├── NPC_REFRESH_CONTENT_CONF.json
│   ├── NPC_REFRESH_RULE_CONF.json
│   ├── SCENE_OBJECT_CONF.json
│   └── WORLD_MAP_BLOCK_CONF.json
├── maps
│   ├── 10003.webp
│   └── 10018.webp
```

## 本地运行

```bash
git clone https://github.com/pathfinderc/rocopath.git
cd rocopath
uv sync
uv pip install -e .
uv run python -m rocopath.main
```
